## Prerequisites

You need Ruby (3.0+) and Bundler installed.

```bash
ruby -v      # should be 3.0+
gem install bundler
```

## Installation

Clone the repo and install dependencies:

```bash
cd seek-dev-notes
bundle install
```

## Running locally

```bash
bundle exec jekyll serve
```

The site will be available at `http://localhost:4000`. Jekyll watches for file changes and rebuilds automatically — just refresh your browser.

## Adding a new doc

1. Create a Markdown file inside `_docs/`:

   ```
   _docs/my-new-doc.md
   ```

2. Add front matter at the top:

   ```yaml
   ---
   title: My New Doc
   description: A short summary shown on the index card.
   category: Guide
   ---
   ```

3. Write your content in standard Markdown below the front matter.

That's it. Jekyll picks up the file automatically and adds it to the sidebar and index.

## Categories

The `category:` field in front matter controls how docs are grouped on the index page and in the sidebar. Use any string — docs with the same category are grouped together, sorted alphabetically.

Common categories used in this site:

| Category | Purpose |
|----------|---------|
| Guide | Step-by-step walkthroughs |
| Reference | API / config / CLI reference |
| Architecture | System design and decisions |

## Search

A search box is in the top header. It searches across title, description, category, and the full content of every doc. Results appear as a dropdown with a highlighted snippet showing where the match was found.

Search is entirely client-side — no server or index required. It loads `search.json` (generated by Jekyll) on first use.

## Building for production

```bash
JEKYLL_ENV=production bundle exec jekyll build
```

Output goes to `_site/`. Deploy that directory to any static host (Nginx, S3, GitHub Pages, Netlify, etc.).

> **Tip:** Add `_site/` to `.gitignore` so the build output doesn't get committed.
