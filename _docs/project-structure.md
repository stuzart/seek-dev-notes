## Directory overview

```
seek-dev-notes/
├── _config.yml          # Jekyll configuration
├── Gemfile              # Ruby gem dependencies
├── index.html           # Home page — auto-generated doc index
├── search.json          # Search data (generated by Jekyll at build time)
│
├── _docs/               # All documentation files live here
│   ├── getting-started.md
│   └── project-structure.md
│
├── _layouts/            # Page templates
│   ├── default.html     # Base HTML shell (header, sidebar, footer)
│   └── doc.html         # Wraps doc content with title and category badge
│
└── assets/
    ├── css/
    │   └── main.css     # All styles — single file, no preprocessor
    └── js/
        └── search.js    # Client-side search logic
```

## Key files

### `_config.yml`

Declares the `docs` collection, sets permalink structure to `/docs/:name/`, and applies the `doc` layout to every file in `_docs/` by default. Change the `title` and `description` fields here to rebrand the site.

### `_docs/`

Drop any `.md` file here and it becomes a doc page. Every file needs this front matter:

```yaml
---
title: Your Page Title
description: One-line summary (shown on index cards).
categories: [Guide, Reference]
---
```

`categories` is a YAML array — a doc can belong to multiple categories and will appear under each one. No other configuration needed — the sidebar, index, and search all update automatically.

### `_layouts/default.html`

The outer shell rendered for every page. Contains:

- Fixed top header with site title and search input
- Left sidebar with navigation grouped by category
- Main content area

Sidebar navigation is built with Liquid: it groups `site.docs` by `category`, sorts alphabetically, and highlights the current page.

### `_layouts/doc.html`

A thin wrapper that adds the category badge and `h1` title above the Markdown content. It extends `default.html` via `layout: default`.

### `search.json`

A Liquid template (despite the `.json` extension) that outputs a JSON array of all docs with their title, category, description, URL, and stripped content. Consumed by `search.js` at runtime.

### `assets/js/search.js`

Fetches `search.json` once on first keystroke, then filters in-memory on every input event. Results are debounce-free (fast enough without it). Minimum query length is 2 characters. Closes on `Escape` or click-outside.

### `assets/css/main.css`

All CSS in one file. Uses CSS custom properties (`--accent`, `--border`, etc.) defined on `:root` for easy theming. No framework, no preprocessor.

## How the index page works

`index.html` uses this Liquid snippet to group and render docs:

```liquid
{% raw %}{% assign docs_by_category = site.docs | group_by: "category" | sort: "name" %}
{% for category in docs_by_category %}
  <h2>{{ category.name }}</h2>
  {% for doc in category.items %}
    <a href="{{ doc.url }}">{{ doc.title }}</a>
  {% endfor %}
{% endfor %}{% endraw %}
```

No plugin needed — just Jekyll's built-in `group_by` filter.

## Adding a new category

Categories are implicit — just use a new string in the `category:` front matter field of any doc. The index and sidebar will include it automatically on the next build.

## Theming

Edit the CSS custom properties at the top of `assets/css/main.css`:

```css
:root {
  --accent: #2563eb;       /* primary blue */
  --accent-light: #eff6ff; /* light tint for hover / backgrounds */
  --text: #1e293b;
  --bg-sidebar: #f8fafc;
  /* … */
}
```
